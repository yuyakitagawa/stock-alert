# bear バックテスト検証

`tools/backtest.py bear`（2024年8月の円キャリー崩壊期＝下落相場）でモデルの暴落耐性を確認し、結果を要約する。CLAUDE.md のテスト基準モード。

## 実行

venv の python を絶対パスで呼ぶこと（CLAUDE.md / メモリ規律）。

```bash
/Users/kitagawayuuya/stock-alert/venv/bin/python3 tools/backtest.py bear 2>&1 | tee logs/backtest_bear_$(date +%Y%m%d).log
```

ネットワークから全銘柄を取得するため数分かかる。途中の `i/N 取得済み...` は正常。

### よく使うオプション

- `--top-n N` … 上位N銘柄を「買い」として検証（デフォルト5）
- `--net-min 15` … ネットスコア最低閾値
- `--compare` … 保有数×閾値を一括比較
- `--no-screener` … スクリーナーをスキップして全銘柄をモデル評価

例: `python3 tools/backtest.py bear --compare`

## 結果の読み方

出力末尾の集計を拾う:

- **平均リターン / 中央値リターン**
- **勝率**（+0%以上の割合）
- **大勝率**（+閾値以上の割合）
- **vs 日経225 → アルファ**（対ベンチマーク超過リターン）

下落予測の精度を重視する（CLAUDE.md: 下落AUC 0.791 > 上昇AUC 0.663）。

## マージ可否の判定（改善マージ規律）

変更の効果を測る場合は **変更前後で bear を実行して数値比較**する。

> 平均リターン・勝率・大勝率のいずれも改善しない変更はマージ禁止（CLAUDE.md §4）。

3指標のうち1つも改善していなければ「マージ不可」と明言し、改善した指標があれば数値を添えて報告する。採用・不採用が決まったら実験コードはその場で削除する（CLAUDE.md §7）。
