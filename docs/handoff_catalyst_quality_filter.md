# 実装依頼: カタリスト候補スクリーンに「利益の質＋本業成長」フィルターを追加

対象セッション: **モデル開発セッション（DB有・ローカル/GitHub Actions環境）**
起票: 2026-06-17（株相談セッションより引き継ぎ）
関連: `tools/screen_catalyst_candidates.py` / `docs/progress_garp_qualitative.md`

---

## 1. 背景・問題（なぜやるか）
現行のカタリスト候補スクリーン（`PBR<1.0 × ROE<8% × 自己資本比率>50%`）は
「資産は厚いが低リターンで割安放置」の銘柄を出すが、**低ROEの"理由"を問うていない**ため、
上位が次の2タイプの"地雷"で埋まる（実際に上位14銘柄を人手レビューして確認済み）:

1. **お化粧決算**: 営業利益は減益なのに、投資有価証券売却益など**一過性の特別利益で純利益を水増し**。
   例) 日本甜菜製糖(2108)・EIZO(6737)。営業益↓×純利益↑のパターン。
2. **主力事業のシュリンク**: 本業が構造的に斜陽で低ROE。
   例) 東邦ガス(9533・規制公益)・大平洋金属(5541・フェロニッケル斜陽)・テレビ局。

→ スクリーンが「カタリストの"起きやすさ（安い箱）"」しか見ておらず、
**"利益の質"と"本業の方向性"を区別できない**のが根本原因。

## 2. ゴール
人手で弾いていた「化粧」「斜陽」を**機械的に除外/減点**し、
「**安い × 本業が伸びている**」候補（例: 飯田G(3291)・日本製鉄(5401)型）だけを残す。

## 3. 実装する2フィルター（既存データで実装可能）

### フィルターA: 利益の質（お化粧除外）
- データ源: `tools/_kabutan_earnings_cache.json`（年次: `revenue/op_profit/ord_profit/net_income`, `is_forecast`）。
  **営業利益はkabutanキャッシュから取る**こと。
- ルール案（直近の実績期 is_forecast=false）:
  - `op_profit <= 0`（営業赤字） → **除外**。
  - `net_income > op_profit × 1.5` かつ `op_profit>0` → 一過性益で水増しの疑い → **減点 or 除外**。
  - 営業利益率 `op_profit/revenue` が直近3期で連続低下 → **減点**。

### フィルターB: 本業の方向性（シュリンク除外）
- データ源: 同上（kabutanキャッシュの年次 op_profit / revenue）＋ 会社予想は `jquants_fin_summary.fop/fsales`。
- ルール案:
  - 直近実績 `op_profit` の前年比 ≤ 0（減益） → **除外 or 強い減点**。
  - 売上の直近3期 CAGR < 0（縮小） → **減点**。
  - 会社予想（`fop`>前期OP, `fsales`>前期売上）で増収増益方向 → **加点**。

### スコア再設計案
現行 `score = (1 - pbr) * equity_ratio` を、
1. フィルターA/Bの**ゲート（除外）**を先に通し、
2. 残った候補に `growth_bonus`（op YoY・売上トレンド・予想方向）を加味、
の二段構えにする。重みは下記§5の検証で決める。

## 4. （別フェーズ・本命）トリガー検知
安い箱を出すだけでは不十分。`lib/alt_data.py` の TOB/増配/自社株買い/大量保有イベントと突合し、
「**構造的候補 × 実際に動きが出ている**」だけに絞るのが本来のエッジ。
EDINET大量保有スキャン実装はそのフェーズ。今回のA/Bはその前段の品質向上。
