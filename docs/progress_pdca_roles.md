## 背景
週次レビューが各ロールの不全を自己診断していたが、`feedback.md` の「必ず improve」指示が
それを上書きし、Quant が XGBoost ハイパラを毎日逆方向に振る「振り子」状態だった。
バックテスト（5期間・少ラウンド）のノイズ範囲（±1〜2%）の上下に反応していた。

## 方針
- **B: ロールを実際に機能させる** — 振り子を止める / FMを判断主体にする
- **C: 本質に向き合う** — ノイズに反応しない / ハイパラ依存から脱却
