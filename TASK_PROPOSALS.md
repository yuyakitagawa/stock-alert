# 改善タスク提案（コードベース確認結果）

## 1) 入力ミスを修正するタスク
- **タスク名**: `tests/test_screener.py` のコメント誤字修正（「モメンタム」→「モメンタム」など文言統一）
- **背景**: テスト内コメントに「スクリーン候補」のようにドメイン用語が揺れており、レビュー時に入力ミス인지仕様語인지判別しづらい。
- **実施内容**:
  - テストコード中の日本語コメントと docstring を README 用語に合わせて統一。
  - しきい値説明（例: 0.05=+5%）を併記して誤読を防止。
- **完了条件**: コメント・docstring の語彙が README と整合し、曖昧な誤記が消えていること。

## 2) バグを修正するタスク
- **タスク名**: `alert_email.py` の銘柄コード変換での例外発生バグ修正
- **背景**: `_row_code_str` は `int(row["銘柄コード"])` を前提としており、空値・非数値・先頭ゼロ付き異常値で `ValueError` が発生し得る。
- **実施内容**:
  - `_row_code_str` を安全変換（`pd.isna` / 文字列正規化 / 例外時スキップ）に変更。
  - 呼び出し側で `None` を許容し、候補抽出処理が落ちないようにする。
- **完了条件**: 不正コードを含む ranking CSV でも `alert_email.py` がクラッシュせず、スキップ処理できること。

## 3) コメント/ドキュメントの矛盾を修正するタスク
- **タスク名**: README と実装の決算除外日数の不整合解消
- **背景**: README では「14日以内に決算がある銘柄に警告」と説明される一方、実装は `CANDIDATE_EARNINGS_SKIP_DAYS = 7` で新規候補を除外している。
- **実施内容**:
  - 仕様を確定（7日 or 14日）し、README と `alert_email.py` のコメントを一致させる。
  - 必要に応じて定数名と説明文を分ける（警告表示日数 vs 新規候補除外日数）。
- **完了条件**: ドキュメントとコードのルールが同じ意味で記述されていること。

## 4) テストを改善するタスク
- **タスク名**: `alert_email.py` の異常データ耐性テスト追加
- **背景**: 既存テストは正常系中心で、ranking CSV の欠損/型崩れに対する回帰防止が弱い。
- **実施内容**:
  - `tests/test_alert_email.py` に以下を追加:
    - 銘柄コードが空/文字列/NaN の行を含むケース
    - `ネット(%)` 欠損時のフォールバック挙動
    - `下落確率(%)` が文字列混在でも候補選定が継続すること
  - 期待結果を「例外を出さない」「候補数が妥当」で検証。
- **完了条件**: 異常入力ケースのテストが追加され、`python3 tests/test_alert_email.py` が通ること。
